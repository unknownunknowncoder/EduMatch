查看详情功能测试

测试说明

这个页面用于测试后台管理系统中"查看详情"按钮的功能。

预期行为：
  • 点击查看详情按钮应该在新窗口中打开对应的详情页面
  • 详情页面应该能够正确加载并显示数据
  • 所有三种类型的详情（帖子、学习计划、资源）都应该能正常工作

1. 后台管理页面测试

点击以下按钮进入后台管理系统，然后测试各个标签页的查看详情功能：

打开后台管理页面
测试步骤：
  1. 进入后台管理页面
  2. 切换到"内容管理"标签，点击任一帖子的查看详情按钮
  3. 切换到"学习计划"标签，点击任一计划的查看详情按钮
  4. 切换到"资源管理"标签，点击任一资源的查看详情按钮
  5. 确认所有详情页面都能在新窗口中正确打开

2. 直接访问详情页面测试

测试直接访问详情页面的URL是否正常工作：

测试帖子详情页面 测试学习计划详情页面 测试资源详情页面
预期结果： 即使ID不存在，页面也应该能够正常加载并显示"未找到"或加载状态，而不是出现500错误。

3. 路由配置检查

确认路由配置中包含以下路由：

  • /post/:id → PostDetail.vue
  • /study-plan/:id → StudyPlanDetail.vue
  • /resource/:id → ResourceDetails.vue
  • /admin → AdminDatabase.vue

4. 服务方法检查

确认 SupabaseService 中包含以下方法：

  • getPostById(id: string)
  • getStudyPlanById(id: string)
  • getResourceById(id: string)
