教育资源对比 AI 推荐工具 MVP 开发计划书
（聚焦免费 / 低价教育资源，低反爬、高落地性）
一、项目概述
（一）项目名称
教育资源对比 AI 推荐工具（MVP 版）
（二）核心定位
聚焦学生、职场人 “找教育资源难、选不准” 的痛点，通过采集官方及垂直平台的免费 / 低价教育资源（课程、考证资料等），结合 AI 需求匹配与评分，提供 “关键词 + 标签输入→AI 筛选→Top3 推荐” 的极简服务，降低学习资源选择成本。
（三）MVP 核心目标
3 周内完成可上线的 MVP 版本，实现 “多平台教育资源采集 + 用户需求匹配 + AI Top3 推荐” 核心功能，验证产品可用性与用户需求，为后续迭代奠定基础。
（四）开发团队与分工
1.	成员 A：负责资源采集模块开发、n8n 工作流搭建、Supabase 数据库设计与维护
2.	成员 B：负责 AI 功能集成、前端开发、测试与 Netlify 部署上线
3.	AI 工具：辅助代码生成（如资源采集脚本、前端组件）、提示词优化（需求拆解与评分逻辑），核心工具：扣子大模型、ChatGPT
二、MVP 功能范围（严格限定，不扩展非核心功能）
（一）用户端核心功能
1.	需求输入：支持教育资源关键词输入（如 “Python 零基础课程”“会计初级考证资料”）；提供标签选择（学习目标：考证 / 技能提升、资源时长：10 小时内 / 10-30 小时 / 30 小时以上、资源类型：免费 / 低价），支持 1-2 个标签勾选 + 50 字内自定义需求输入。
2.	资源展示：展示采集到的教育资源基础信息，包括资源名称、提供方（如中国大学 MOOC、B 站 UP 主）、内容大纲（3-5 条核心知识点）、学习时长、用户评分（平台公开评分）、获取链接（直达资源页面）。
3.	AI 推荐：按 “需求匹配度 60%+ 内容质量 25%+ 学习成本 15%” 生成综合得分，推荐 Top3 资源，并标注核心契合点（如 “匹配‘Python 零基础’需求，免费无广告，适合新手入门”）。
（二）技术端核心功能
1.	资源采集模块：采集 4 类平台资源，字段包括资源名称、提供方、内容大纲、学习时长、用户评分、获取链接、资源类型（课程 / 资料）：
1.	官方教育平台：国家中小学智慧教育平台（免费课程）、中国大学 MOOC（公开课程，调用官方 API）；
2.	垂直教育平台：B 站知识区（公开课程，采集标题、时长、播放量、评分）、网易云课堂免费专栏；
3.	考证资源平台：人社局官方考证指南（报考条件、备考资料链接）、会计协会免费备考文档。
1.	数据处理模块：实现资源去重（按 “资源名称 + 提供方” 唯一标识）、内容大纲提炼（提取核心知识点，精简至 3-5 条）、用户评分标准化（统一按 10 分制换算，如 B 站 5 星→10 分、MOOC4.5 分→9 分）、资源类型分类（自动标注 “课程”“资料”）。
2.	工作流模块：通过 n8n 串联 “用户需求触发→多平台资源采集（API 调用 + 轻量采集）→数据清洗→Supabase 存储→AI 分析→结果返回” 全流程，支持定时更新资源（每日凌晨更新 1 次，保证资源时效性）。
3.	AI 分析模块：需求拆解（将用户 “关键词 + 标签 + 自定义需求” 转化为可量化维度，如 “Python 零基础”→“包含 Python 基础语法、无编程经验门槛”）；资源综合评分（按权重计算得分，需求匹配度看资源是否覆盖用户核心需求，内容质量参考用户评分与提供方权威性，学习成本结合是否免费、时长合理性）。
（三）非核心功能暂不开发
用户注册登录、资源收藏 / 历史查询、资源评论互动、付费资源推荐、复杂数据可视化（如学习时长趋势图），留待后续迭代。
三、技术选型（适配低反爬、低成本）
（一）前端技术栈
4.	框架：Vue3 + Vite（开发效率高，热更新快，适合快速迭代）
5.	UI 组件库：Element Plus（轻量化，提供表单、卡片等基础组件，适配资源展示与需求输入场景）
6.	数据请求：Axios（处理与 n8n 工作流的接口对接，获取资源与推荐结果）
7.	部署平台：Netlify（免费额度满足 MVP 需求，支持 CI/CD 自动部署，无需额外服务器成本）
（二）后端与资源采集技术栈
8.	资源采集：Python（轻量采集脚本，针对 B 站、网易云课堂等无反爬平台）、中国大学 MOOC 官方 API（直接调用，无需爬虫）；无需复杂反爬策略，仅需设置采集间隔（每平台每次采集间隔 2-3 秒，避免高频请求）
9.	工作流工具：n8n 云版（无需本地部署，拖拽式配置流程，支持 API 调用、数据清洗、数据库写入节点，快速串联全流程）
10.	数据库：Supabase（开源 Firebase 替代方案，内置 PostgreSQL 数据库 + REST API，无需单独搭建后端，支持资源数据存储与查询，免费额度 500MB 满足 MVP 需求）
（三）AI 技术栈
11.	大模型：扣子大模型 API（无需本地部署，调用便捷，支持中文场景下的需求拆解与评分逻辑，免费额度满足 MVP 调用量）
12.	提示词工程：自定义场景化提示词，聚焦 “教育资源需求拆解”（如 “将用户输入的‘会计初级考证资料’需求，拆解为‘包含初级会计实务、经济法基础知识点，有真题解析，适合 3 个月备考周期’”）与 “资源评分”（如 “根据资源是否覆盖拆解后的需求点、用户评分、是否免费，按权重计算综合得分”）。
四、开发阶段规划（3 周周期）
（一）第一阶段：需求拆解与基础搭建（1 周）
1.	确定资源采集字段（如资源名称、提供方等 8 个核心字段）、用户输入标签清单（学习目标、时长、类型），完成技术栈最终确认；
2.	搭建 Supabase 数据库（设计 “教育资源表”“用户需求记录表”，配置数据读写权限，测试 REST API 调用）；
3.	初始化前端项目（Vue3+Vite+Element Plus），开发需求输入（表单）、资源展示（卡片）、AI 推荐（结果列表）页面骨架，关联 Netlify 实现自动部署；
4.	完成中国大学 MOOC API 调用测试，开发 B 站、网易云课堂轻量采集脚本，测试单平台资源采集与写入 Supabase。
（二）第二阶段：全流程开发与集成（1 周）
1.	开发数据处理模块（去重、评分标准化、大纲提炼），集成到资源采集流程；
2.	搭建 n8n 工作流，串联 “用户需求接收→多平台资源采集（API + 脚本）→数据清洗→Supabase 存储→AI 分析调用→结果返回前端” 全流程；
3.	编写 AI 核心提示词（需求拆解、资源评分），完成扣子大模型 API 对接，实现 AI 推荐逻辑；
4.	完成前端与 n8n、AI 接口的联调，实现 “需求输入→资源展示→AI 推荐” 端到端功能。
（三）第三阶段：测试优化与部署上线（1 周）
1.	进行功能测试（核心流程通畅性、资源采集准确性、AI 推荐合理性）、兼容性测试（主流浏览器：Chrome、Edge、Safari）；
2.	优化细节：调整前端页面响应速度（如资源卡片懒加载）、完善 AI 推荐提示语（更清晰的契合点描述）、补充异常处理（如资源链接失效时提示 “该资源暂不可用”）；
3.	配置 Supabase 自动备份（每日备份数据库，防止数据丢失），整理技术文档（资源采集脚本说明、n8n 工作流配置、AI 提示词）、用户操作手册（极简版，3 步说明使用流程）；
4.	前端最终部署到 Netlify，邀请 10-20 名目标用户（学生、职场人）试用，收集反馈并进行紧急优化，完成 MVP 上线。

六、风险与应对措施

风险类型	具体风险	应对措施

进度风险	某模块开发延迟（如工作流配置复杂）	优先开发核心流程（需求输入→资源采集→AI 推荐），非核心功能（如定时更新）后续补充；每日短会同步进度
七、验收标准
（一）功能验收
1.	核心流程通畅：用户输入需求后，10 秒内返回资源展示与 AI Top3 推荐结果；
2.	资源准确性：采集的资源信息（名称、提供方、链接）与原平台一致度≥95%，无重复资源；
3.	AI 推荐合理性：试用用户认可推荐资源符合需求的比例≥80%；
4.	稳定性：连续 48 小时无人工干预，资源采集成功率≥90%，工作流无崩溃。
（二）性能验收
1.	前端加载：首屏加载≤2 秒，资源列表加载≤3 秒；
2.	并发支持：5 人同时访问，无页面卡顿、接口超时；
3.	数据库性能：Supabase 资源查询响应时间≤300ms。
（三）文档验收
1.	技术文档：包含资源采集脚本（Python 代码及说明）、n8n 工作流配置截图与步骤、AI 提示词（需求拆解 + 评分逻辑）；
2.	用户手册：1 页极简版，包含 “输入关键词→选标签→看推荐”3 步操作说明，附示例截图。
八、后续迭代规划（MVP 上线后）
（一）短期迭代（1-2 个月）
1.	功能扩展：增加用户注册登录（基于 Supabase Auth）、资源收藏与历史查询；
2.	资源丰富：新增更多垂直平台（如腾讯课堂免费课、知乎盐选低价课）；
3.	AI 优化：基于用户反馈调整评分权重，支持 “资源难度” 标签（入门 / 进阶 / 高级）。
（二）中期迭代（3-4 个月）
1.	个性化推荐：基于用户历史查询与收藏，推荐更匹配的资源；
2.	互动功能：支持用户对资源进行评论、打分，补充 “真实使用体验”；
3.	多端适配：开发 H5 版本，适配手机端浏览。
